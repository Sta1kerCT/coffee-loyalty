# Техническое задание: Система лояльности кофейни

## 1. Тема

Кофейня (система лояльности): простая бизнес-логика начисления и списания для баллов

---

## 2. Функциональные требования

- **Клиенты:** регистрация, просмотр и редактирование профиля (ФИО, телефон, email), на валидацию забиваем, не хочу мучиться с регулярками. У каждого клиента баланс баллов.
- **Транзакции баллов:** начисление за покупку (положительное целое число), списание за вознаграждение (целое отрицательное). Каждая операция фиксируется с датой и каиндом PURCHASE/REWARD.
- **Награды:** название и стоимость в баллах. Клиент может обменять баллы на награду при достаточном балансе.
- **Покупки:** клиент, сумма чека, начисленные баллы. Начисление баллов по правилу (например, 1 балл = 10 руб.).

---

## 3. Основные сущности

- **Client** — клиент: id, ФИО, телефон, email, дата регистрации, баланс баллов
- **PointTransaction** — транзакция баллов: id, client_id, amount (+/-), type, created_at
- **Reward** — награда: id, name, points_cost
- **Purchase** — покупка: id, client_id, amount_rub, points_earned, created_at

---

## 4. Нефункциональные требования

- REST API для всех операций с клиентами, транзакциями, наградами и покупками.
- Хранение данных в СУБД (PostgreSQL/H2).
- Асинхронная отправка целочисленных значений изменений баллов в топик Kafka для последующей загрузки в ClickHouse.
- Развёртывание: Docker / Kubernetes, CI/CD (GitLab CI или Jenkins).
- Мониторинг: метрики в Grafana, алерты при ошибках в логах и при недоступности сервиса.
- Автотесты: API (Postman/коллекция), UI (Selenium/Selenide).
